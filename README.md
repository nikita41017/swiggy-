# 📊 Swiggy Sales Data Analysis — From Data to Decisions

## 🚀 Overview

This project focuses on analyzing Swiggy sales data to extract **actionable business insights** rather than performing surface-level exploratory analysis.

The objective is to simulate a real-world scenario where data is used to support **operational efficiency, demand planning, and strategic decision-making**.

---

## 🎯 Problem Statement

Food delivery platforms operate in highly dynamic environments where demand fluctuates across time, location, and category.

This project aims to answer:

* When does demand peak, and how can operations adapt?
* Which categories consistently drive revenue?
* What customer behavior patterns can inform business strategy?

---

## 🧠 Analytical Approach

The analysis is structured around **business-centric thinking**:

### 1. Data Preprocessing

* Handling missing values and inconsistencies
* Data type corrections and formatting
* Preparing dataset for analysis

### 2. Exploratory Data Analysis (EDA)

* Temporal analysis of orders
* Category-wise performance evaluation
* Revenue distribution patterns

### 3. Insight Generation

* Identification of peak demand windows
* High-performing vs. low-performing segments
* Behavioral trends in ordering patterns

---

## 📈 Key Insights

* Clear identification of **peak ordering hours**
* Certain categories contribute disproportionately to revenue
* Detectable patterns in customer ordering behavior

These insights can support:

* Demand forecasting
* Resource allocation
* Business optimization strategies
